## 기술 스택 결정

### 핵심 판단 기준
- 캔버스 위 오브젝트 자유 편집 (이동/크기/회전) 필요
- 텍스트 인라인 편집 필요
- PNG/JPG 내보내기 필요
- 별도 서버 없이 프론트엔드만으로 동작

### 선택 스택

| 역할 | 선택 | 이유 |
|------|------|------|
| 캔버스 엔진 | **Fabric.js** | 오브젝트 기반, 선택/이동/회전 내장, 텍스트 편집 지원, 문서 풍부 |
| UI 프레임워크 | **Vanilla JS + HTML/CSS** | 의존성 최소화, Fabric.js와 충돌 없음 |
| 내보내기 | **Fabric.js 내장 toDataURL** | 별도 라이브러리 불필요 |
| 폰트 | **Google Fonts (CDN)** | 한국어 폰트 무료 제공 |
| 저장 | **localStorage** | 서버 없이 자동저장 |

### 파일 구조

```
product_page/
├── index.html          # 템플릿 갤러리 (시작 화면)
├── editor.html         # 에디터 화면
├── css/
│   ├── gallery.css
│   └── editor.css
├── js/
│   ├── gallery.js      # 템플릿 갤러리 로직
│   ├── editor.js       # 에디터 메인 로직
│   ├── canvas.js       # Fabric.js 캔버스 제어
│   ├── blocks.js       # 블록 정의 및 렌더링
│   ├── themes.js       # 테마(색상+폰트) 정의
│   ├── toolbar.js      # 상단 툴바 (저장/내보내기 등)
│   ├── panel-left.js   # 좌측 패널 (요소 추가, 블록 추가)
│   ├── panel-right.js  # 우측 패널 (선택 요소 속성)
│   └── export.js       # PNG/JPG 내보내기
├── data/
│   ├── templates.js    # 템플릿 정의 (블록 조합)
│   └── blocks.js       # 블록 라이브러리 정의
└── assets/
    └── placeholder/    # 플레이스홀더 이미지
```

## 기술적 핵심 구현 포인트

### 출력 영역만 내보내기
```
전체 캔버스가 아닌 아트보드 영역만 크롭해서 저장
→ Fabric.js toDataURL()의 left/top/width/height 옵션 활용
```

### 블록 세로 쌓기 구조
```
블록은 Fabric.js 오브젝트 그룹이 아닌 논리적 단위로 관리
→ 각 블록의 Y 오프셋을 계산하여 요소 배치
→ 블록 순서 변경 시 관련 요소 Y좌표 일괄 재계산
```

### 테마 일괄 적용
```
캔버스 내 모든 텍스트/도형 오브젝트를 순회하며
→ 오브젝트 타입별로 테마 색상/폰트 매핑하여 일괄 변경
```
